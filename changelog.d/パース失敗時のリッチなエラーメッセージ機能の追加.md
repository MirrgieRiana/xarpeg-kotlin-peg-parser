# パース失敗時のリッチなエラーメッセージ機能の追加

## 追加

### `ParseException.formatMessage()` 拡張関数

パース失敗時にユーザーフレンドリーなエラーメッセージを生成する `formatMessage` 拡張関数を追加しました。この関数は以下の情報を含むリッチなエラーメッセージを提供します：

- エラーの行番号と列番号
- 期待される名前付きパーサのリスト（利用可能な場合）
- エラーが発生した行のソースコード（80文字を超える場合は自動的に省略）
- エラー位置を示すキャレット（`^`）記号

例：

```kotlin
try {
    parser.parseAllOrThrow(input)
} catch (exception: ParseException) {
    val message = exception.formatMessage()
    check(message[0] == 'E')
    check(message.contains("line 1"))
}
```
